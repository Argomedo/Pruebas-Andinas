# Contexto del negocio

Distribuidora Andina SpA es una empresa chilena que vende productos de consumo masivo (bebidas, snacks y alimentos) a minimarkets, restaurantes y almacenes de barrio.

## Situación actual

Actualmente el proceso de ventas es manual:
- Los clientes hacen pedidos por WhatsApp o teléfono
- Un ejecutivo registra el pedido en una planilla Excel
- Bodega prepara el pedido
- Se coordina el despacho
- El cliente paga al recibir el producto

Este proceso genera múltiples problemas:
- Errores en pedidos
- Retrasos en entregas
- Pérdida de pedidos
- Mala experiencia de clientes
- Falta de control de inventario
- Dificultad para escalar el negocio

## Proyecto de Transformación Digital

La empresa decide desarrollar una plataforma digital de gestión de pedidos que permita automatizar el proceso completo.

La plataforma tendrá tres componentes principales:

### 1. Portal de Clientes
Aplicación web donde los clientes pueden:
- Registrarse
- Ver catálogo de productos
- Realizar pedidos
- Revisar historial de compras
- Ver estado de sus pedidos
- Descargar facturas

### 2. Panel Administrativo
Sistema utilizado por la distribuidora para:
- Gestionar pedidos
- Administrar clientes
- Administrar productos
- Controlar inventario
- Gestionar despachos
- Generar reportes de ventas

### 3. Aplicación para Repartidores
Aplicación móvil para conductores que permite:
- Ver pedidos asignados
- Ver rutas de entrega
- Confirmar entrega
- Registrar incidencias
- Capturar firma del cliente

## Problemas que deben evitarse (contexto de calidad)
El sistema debe evitar situaciones como:
- Pedidos duplicados
- Productos vendidos sin stock
- Pedidos que no llegan a bodega
- Errores en facturación
- Caídas del sistema en horario peak
- Lentitud en la plataforma
- Accesos no autorizados
- Pérdida de información
